Validation Design: Weight Sum Tolerance
Needs Review Design Decision
The ticket description contains two different validation rules. The acceptance criteria state "weights sum to 100" while the description body says "99-100" and also references "+/-1%". These are three different tolerances. The existing weighted_choice() in helpers/functions.php may handle sub-100 sums by assigning leftover weight to the last entry, or it may not. This needs to be clarified before implementation to avoid a validation rule that rejects valid configs or accepts invalid ones.
